# Planes de los 3 frentes inmediatos de ReWind

Planes diferenciados (table stakes vs. diferenciadores) para los tres frentes que
trabajamos primero, todos apoyados en el modelo 4D georreferenciado. Ver el
estado/numeración en [`../ROADMAP.md`](../ROADMAP.md).

| Frente | Ítem | Plan |
|---|---|---|
| Etapas constructivas / avance de obra (HUD tipo Stark + dashboard de parque) | `R-18` | [frente-1-avance-obra.md](frente-1-avance-obra.md) |
| Análisis de sombras (shadow flicker) | `R-19` | [frente-2-sombras.md](frente-2-sombras.md) |
| Gemelo de construcción (f₁ predicha-vs-medida por etapa) | `R-31` | [frente-3-gemelo-construccion.md](frente-3-gemelo-construccion.md) |

**Hilo conductor del diferenciador:** ReWind es el único que junta **gemelo físico +
flota georreferenciada 3D + construcción y operación** en un mismo navegador. Los
competidores son piezas sueltas (DroneDeploy para avance, WindPRO para sombras,
Wölfel para SHM).

> Estado: **planes para discutir/iterar**. Las decisiones acordadas se reflejan en
> cada archivo (sección «Decisiones»). Al implementar, se cierra el ítem `R-*` en el
> roadmap.
